完全不需要提前计算贡献啊，为什么题解都是。

原题面中的 kkk 在这里用 mmm 表示。

将 aaa 排序后，显然有 DP 设 fi,j,kf_{i, j, k}fi,j,k​ 为考虑前 iii 个，有 jjj 组还没有放 max⁡\maxmax，当前不和谐度之和为 kkk 的方案数。转移是显然的，每次加入一个数考虑它是新一组的唯一元素 / 新一组的 min⁡\minmin / 之前一组的中间值 / 之前一组的 max⁡\maxmax 四种情况：

fi,j,k=(j+1)(fi−1,j,k+fi−1,j+1,k−ai)+fi−1,j−1,k+aif_{i, j, k} = (j + 1)(f_{i - 1, j, k} + f_{i - 1, j + 1, k - a_i}) + f_{i - 1, j - 1, k + a_i} fi,j,k​=(j+1)(fi−1,j,k​+fi−1,j+1,k−ai​​)+fi−1,j−1,k+ai​​

直接这样转移，第三维中途可能会很小 / 很大，所以时间复杂度为 O(n2∑ai)O(n^2\sum a_i)O(n2∑ai​)。

但我们可以剪枝啊！有很多状态最终是无法转移到最终合法状态的。

考虑一个状态 fi,j,kf_{i, j, k}fi,j,k​ 能转移到的最小不和谐度。显然，将 aia_iai​ 后面的 jjj 个作前面 jjj 组的 max⁡\maxmax，剩下的单独成组即可。令 sm=∑p=i+1i+japsm = \sum_{p = i + 1}^{i + j} a_psm=∑p=i+1i+j​ap​，即最终不和谐度为 k+smk + smk+sm。则 k+sm≤mk + sm \le mk+sm≤m 时，fi,j,kf_{i, j, k}fi,j,k​ 才是有用的。

将不等式变形，得到 k≤m−smk \le m - smk≤m−sm，这是 kkk 的一个上界。而 kkk 的下界 lklklk 显然是 −∑p=i−j+1iap-\sum_{p = i - j + 1}^i a_p−∑p=i−j+1i​ap​（在前 iii 个中选最后 jjj 个作为 min⁡\minmin，剩下的单独成组），又因为 aaa 的单调性，所以 −sm≤lk≤k≤m−sm-sm \le lk \le k \le m - sm−sm≤lk≤k≤m−sm（前面 jjj 个加起来肯定小于等于后面 jjj 个），则 m−sm−lk≤m−sm+sm=mm - sm - lk \le m - sm + sm = mm−sm−lk≤m−sm+sm=m，对于一对 i,ji, ji,j，有用的 kkk 只有 O(m)O(m)O(m) 个。

每次只转移有用的 kkk，时间复杂度即可减为 O(n2m)O(n^2m)O(n2m)。main code：

for (int i = 1;i <= n;i ++){swap(f0, f1);
	for (int j = 0;j <= i && j <= n - i;j ++){
		int le = sm[i - j] - sm[i], ri = m - sm[i + j] + sm[i];

			if (j < n) add(f1[j][k], (j + 1ll) * f0[j + 1][k - a[i]] % mod);
			if (i == n && !j) add(ans, f1[j][k]);
		}
