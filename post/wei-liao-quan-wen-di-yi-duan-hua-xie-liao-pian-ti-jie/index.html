为了全文第一段话写了篇题解

2025-01-29 题解

我寻思这种“一正常题非要爆改计数嗯堆难度结果整出一托答辩”不应该是模拟赛恰烂钱才能干的事吗，怎么扔 CF 了。真不如来做我们 P11269 爆改版。

考虑 E1 咋做。先考虑快速算出 SG(a,x)\textrm{SG}(a, x)SG(a,x)。如果直接打表你会寄。所以在函数的原本意义上先做一些操作。可以发现，我们只需要保留 aaa 中 xxx 上为 111 的那些位，其他位的信息我们可以移动到那些位上。

假如 aaa 中有一位为 111 且 xxx 中那一位为 000，那么我们可以将这位扔掉，并将 aaa 在 xxx 中更低的 111 位上都赋为 111。那么 aaa 只在 xxx 中有值的位上有值，并且我们并不关心 xxx 中有值的位具体是哪些，所以我们可以将其平移到最前面，那么 xxx 就变为了 2k−12^k - 12k−1 这样的数。更进一步的，我们可以将 aaa 最高位之后的位都扔掉，那么新的 xxx 最高位和 aaa 的最高位相等，于是 SG(a,x)\textrm{SG}(a, x)SG(a,x) 一定能被转化成 SG(a′,2k−1)\textrm{SG}(a', 2^k - 1)SG(a′,2k−1)，其中 kkk 为 a′a'a′ 最高位，设其为 G(a′)G(a')G(a′)。

那么我们对 GGG 打表，规律就显而易见了：我们有：

G(n)={0n=2k−2 (k>0)k⊕1n=2k⌈log⁡2n⌉otherwiseG(n) = \begin{cases} 0 & n = 2^k - 2 \ (k > 0)\\ k \oplus 1 & n = 2^k\\ \lceil\log_2 n\rceil & \mathrm{otherwise}\\ \end{cases} G(n)=⎩⎪⎨⎪⎧​0k⊕1⌈log2​n⌉​n=2k−2 (k>0)n=2kotherwise​

那么我们就以 O(nlog⁡2v)O(n \log_2 v)O(nlog2​v) 解决了 E1。

考虑解决 E2。由于上述结论，SG\mathrm{SG}SG 值显然是 O(log⁡2v)O(\log_2 v)O(log2​v) 的，那么我们只要算出每堆石子取到每种 SG\mathrm{SG}SG 值的方案数，那么就能 O(nlog⁡22v)O(n \log_2^2 v)O(nlog22​v) 合并出最终的答案。

观察计算 SG\mathrm{SG}SG 值的代码：

inline int SG(int a, int x){bool flg = 0;int s = 0;
	for (int i = 29;i >= 0;i --){
		int aa = a >> i & 1, xx = x >> i & 1;
		if (!xx) flg |= aa;
		else aa |= flg, s <<= 1, s |= aa;
	}
	if ((s + 2 & s + 1) == 0) return 0;
	if ((s & s - 1) == 0) return __lg(s) ^ 1;
	return __lg(s) + 1;
}

我们发现，我们只关注 flg,log⁡2sflg, \log_2 sflg,log2​s 以及 sss 是否为 2k−2,2k2^k - 2, 2^k2k−2,2k 形式，并且我们所做的操作就是根据 aa,xxaa, xxaa,xx 的取值来往 sss 的后面加一位。

那么我们考虑建出一个识别 010101 串的自动机，只需要识别 1000⋯01000\cdots01000⋯0 以及 111⋯10111\cdots10111⋯10 两类串，显然这个自动机只有常数个节点。

这样就可以数位 DP 了：从高到低考虑 xxx 的每一位，那么我们需要记录；

  • 当前考虑到哪一位；
  • 是否顶到 bbb 的上界；
  • flgflgflg 的值；
  • 走到了自动机的哪个节点；
  • 串中第一个 111 与串尾的距离。

显然可以枚举这位选 000 还是 111 来做到 O(1)O(1)O(1) 转移。对于单个数求方案数就是 O(log⁡22v)O(\log_2^2 v)O(log22​v) 的。那么总时间复杂度就是 O(nlog⁡22v)O(n \log_2^2 v)O(nlog22​v)。代码非常好写，E2 喜提 CF 最短解。
