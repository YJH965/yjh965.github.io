给定序列 aaa，并且给出两种操作：

  • 1 x y v：将所有 aia_iai​ 的值加上 vvv，其中 i≡y(mod2x)i\equiv y\pmod {2^x}i≡y(mod2x)。
  • 2 x y：询问所有 aia_iai​ 的和，其中 i≡y(mod2x)i\equiv y\pmod {2^ x}i≡y(mod2x)。

题解

一道结合了 Trie 和 Lazytag 的好题。

我们考虑修改和查询涉及到的下标。

发现，一个数对 2x2^x2x 取模，就相当于取这个数的二进制表示的右 xxx 位。于是，所有的满足 i≡y(mod2x)i \equiv y\pmod {2^x}i≡y(mod2x) 的 iii 都有一个长度为 xxx 的后缀 yyy。

所以，我们考虑把所有的下标的二进制形式倒序插入一颗 01-Trie，那么每次修改就是子树加，查询就是子树查询。

在 01-Trie 中维护 sumsumsum 和 sizsizsiz 即可。

题解

比较小清新的数据结构。

首先考虑限制条件的本质。

设数组 preipre_iprei​ 为使得 j<ij < ij<i 且 aj=aia_j = a_iaj​=ai​ 的最大的 jjj，且当前左右端点分别为 l,rl, rl,r，那么一个数 aia_iai​ 在 [l,r][l, r][l,r] 中只出现了一次当且仅当：

  • 它是 [1,r][1, r][1,r] 中最后一个 aia_iai​。

  • prei<lpre_i < lprei​<l。

  2. 将询问离线下来按照 rrr 排序，只需要普通的线段树。

给出一个 111 到 nnn 的排列，现在对这个排列进行 mmm 次局部排序，排序分为两种，将区间 [l,r][l, r][l,r] 的数字升序或降序排序。

最后询问位置 ppp 上的值。

题解

神仙二分 + 线段树。

我们考虑二分位置 ppp 的值。怎么 check 呢？

我们考虑 check ap≥xa_p \ge xap​≥x。

构造一个 01 序列 bbb，∀i∈[1,n],bi=[ai≥x]\forall i \in [1, n], b_i = [a_i \ge x]∀i∈[1,n],bi​=[ai​≥x]。

然后在 01 序列 bbb 上执行排序操作，最后如果 bp=1b_p = 1bp​=1 则 ap≥xa_p \ge xap​≥x。

为什么这样是对的呢？因为在 bbb 序列中，元素的相对大小顺序没有改变，大于等于 xxx 的元素还是大于小于 xxx 的元素。

排序操作可以用支持区间修改的线段树实现。

给定 nnn 件物品，第 iii 件物品有如下信息：

  • 卖出去可以得到 pip_ipi​ 的收益。

  • 过期时间为 did_idi​，过了过期时间就不能再卖出去。

卖掉一件物品要用 111 的时间，求最大收益。

题解

首先，我们可以想到一个贪心思路：把物品按照 pip_ipi​ 从大到小排序，优先卖出 pip_ipi​ 大的物品，且每一个物品的卖出时间都尽量靠后。

所以对于每一个位置，我们支持占用一个位置，和求出某个位置前面第一个空闲的位置。

我们考虑使用并查集维护。如果占用了位置 ppp，那么把位置 ppp 合并到位置 p−1p - 1p−1 上，查询时查询某个位置的根节点。

给一个长度为 nnn 的字符串，mmm 次询问，每次询问一个连续子串是否可以重排成一个回文串。

题解

一道很不错的莫队题。

显然，一个区间重排后可以成为一个回文串当且仅当其中有 000 或 111 个字符出现了奇数次。

我们只关心字符出现次数的奇偶性，所以我们可以把字符串所有的前缀状压成一个数，查询时直接异或即可。

添加 / 删除时枚举是哪一种字符出现了奇数次，时间复杂度为 O(∣Σ∣nn)O(|\Sigma|n \sqrt n)O(∣Σ∣nn​)。

题解

整块段的众数可以 O(n)O(n)O(n) 预处理，两个散块里的数最多是块长级别，预处理出每一种数出现的位置，在位置数组里二分查找即可解决散块的问题，可以做到单次查询 O(blog⁡n)O(b \log n)O(blogn)。

你需要维护一个长度为 nnn 的 01 序列。

你会接到 qqq 条信息：

  1. 告诉你一个区间的按位或值。
  2. 问你序列中一个数的值。如果不能确定，输出 N/A。

保证信息正确。

题解

我们称按位或为 xxx 的区间为“xxx 区间”。

可以发现一些性质：000 区间里每个数都是 000；111 区间至少有一个数为 111。

能确定一个数为 000，当且仅当一个 000 区间包含这个数；能确定一个数为 111，当且仅当一个 111 区间包含这个数，且该区间内其余的数均被确定为 000。所以，一个 111 区间至多能确定一个数为 111。

我们考虑把操作序列离线下来，这样我们就可以算出每个位置得知所有信息后的值。然后求出每个位置能确定的最早时间即可。

具体来讲，我们记录所有修改操作和询问操作的时间，然后确定每个数最早被确定为 000 的时间。

然后对于每个 111 区间，如果该区间内有 >1> 1>1 个数到最后也未被确定为 000，则不考虑该区间；否则，设唯一的最后也未被确定为 000 的数在位置 ppp，该区间内最后一个被确定为 000 的数在时刻 lstlstlst 被确定为 000，收到该区间的时刻为 ttt，那么位置 ppp 在 max⁡(lst,t)\max(lst, t)max(lst,t) 时刻可以被该区间确定为 111，一个位置能被确定为 111 的时刻显然是最先确定该位置的 111 区间确定该位置的时刻。

所有任务都可以用一棵线段树完成，时间复杂度为 O(nlog⁡n)O(n \log n)O(nlogn)。

有一棵无根树，点数为 nnn，每个点有个点权 aua_uau​，定义一棵树是合法的，当且仅当树上所有简单路径上点权的异或和都不为 000。

你可以对点权进行修改，可以改成任意正整数，问最少修改多少次才能让这棵树合法。

题解

启发式合并。

首先，对于每一个 uuu，我们可以预处理出 111 到 uuu 的简单路径上点权的异或和，记为 dud_udu​。然后 uuu 到 vvv 的简单路径上点权的异或和显然就是 du⊕dv⊕alca⁡(u,v)d_u \oplus d_v \oplus a_{\operatorname{lca}(u, v)}du​⊕dv​⊕alca(u,v)​，我们考虑枚举 lca⁡(u,v)\operatorname{lca}(u, v)lca(u,v) 来修改 alca⁡(u,v)a_{\operatorname{lca}(u, v)}alca(u,v)​。

我们还发现，一个节点如果要修改的话，一定会改成远大于值域的一个极大值，使经过这个点的每一条简单路径的异或和都不为 000。

所以我们考虑这样的一个贪心：对于每一个节点开一个 set 维护以这个节点为根的子树里所有节点 uuu 的 dud_udu​，特别的，如果这个节点修改成了极大值，这个节点对应的 set 为空。然后对于整棵树 dfs 一遍，假设当前在节点 uuu，递归进入每一个儿子 vvv，回溯时将 vvv 的 set 使用启发式合并动态并入 uuu 的 set，并在插入过程中动态查找当前 uuu 的 set 中是否有两个数异或值为 aua_uau​。如果有，aua_uau​ 必须修改成极大值，将 set 清空即可。至于清空的原因，是因为 uuu 的子树里任何一个点都不可能与外面一个点构成一条异或和为 000 的简单路径。

贪心正确性显然，时间复杂度为 O(nlog⁡2n)O(n \log^2 n)O(nlog2n)。
